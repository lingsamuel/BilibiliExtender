# 作者分组管理注入 Spec

## 1. 背景

当前插件的分组能力主要集中在“分组动态”抽屉内，缺少面向站点原生作者入口的操作点，导致用户在浏览视频页、用户空间或头像悬浮卡片时：

- 无法快速判断某个作者是否已经被纳入插件分组管理；
- 无法直接把当前作者加入某个分组；
- 无法直接把某个作者从某个分组移除。
- 无法在“添加到分组”弹框内直接创建新的插件分组；
- 无法在“添加到分组”弹框内直接新建收藏夹并将其绑定为插件分组。

本 Spec 定义“作者分组管理”注入能力，在站点已有“关注”按钮附近补充插件入口，并允许用户围绕作者直接管理分组归属。

## 2. 目标与非目标

### 2.1 目标

- 在作者相关的三个站点原生入口旁注入插件按钮。
- 按按钮当前状态让用户快速判断该作者是否已经属于任一插件分组。
- 点击按钮后弹出分组管理弹框，展示全部插件分组与当前作者的分组归属。
- 支持在弹框中直接执行“加入分组 / 移除分组”。
- 支持在弹框中直接把“尚未绑定的已有收藏夹”创建为插件分组。
- 支持在弹框中直接新建收藏夹，并同步创建对应的插件分组。
- 弹框默认只读取本地缓存的“我的收藏夹列表”快照，不在打开时自动请求远端；当缓存缺失或用户认为数据过旧时，可手动刷新。
- 操作成功后同步更新插件缓存，并补充对应分组的收藏夹刷新请求。

### 2.2 非目标

- 本轮不把“作者分组管理”入口扩展到搜索页、评论区、动态页等其他作者入口。
- 本轮不引入“实时查询收藏夹真相”的强一致模式。
- 本轮不支持手动输入视频 ID 或作者 ID 进行分组操作。
- 本轮不在该弹框内提供分组别名编辑；别名与其他分组管理操作仍留在设置页。

## 3. 注入范围

新增注入点仅覆盖以下页面中的作者入口区域：

1. 视频投稿页：`https://www.bilibili.com/video/<BVID>`
2. 用户空间：`https://space.bilibili.com/<USER_ID>`
3. 用户头像 hover 卡片

统一要求：

- 按钮应尽量贴近站点原生“关注”按钮布局与尺寸语义，但保留插件自身可识别度。
- DOM 注入必须具备幂等性，避免站点局部重渲染时重复插入多个按钮。
- 对于异步渲染与 hover 卡片等瞬态 DOM，需使用稳健的观察与重挂载策略。

## 4. 状态来源

### 4.1 分组归属口径

- 作者是否属于某个分组，只基于插件当前本地缓存判断，不主动实时扫描收藏夹接口。
- 缓存正确性默认由现有调度器后台维护。
- 通过本功能触发的“加入分组 / 移除分组”操作，允许直接同步改写对应分组缓存中的作者归属，减少用户感知延迟。

### 4.2 按钮状态

- 若作者当前不属于任何启用中的插件分组，按钮显示“添加到分组”态。
- 若作者当前至少属于一个插件分组，按钮显示“已分组(N)”态，其中 `N` 为当前作者所属分组数。
- 当 `N=1` 时，按钮文案仍显示为 `已分组(1)`，不退化为无数量版本。
- 进入弹框后再展示具体属于哪些分组。

## 5. 交互设计

### 5.1 入口按钮

- 入口按钮放置在站点原生“关注”按钮附近。
- 在插件抽屉的“近期投稿 / 全部投稿”作者标题行中，也要在现有关注按钮旁提供同一能力入口。
- 点击按钮打开作者分组管理弹框。
- 若当前没有任何插件分组，允许打开弹框，但内容区显示空态提示：
  - 默认仍以“已有插件分组列表”为主视图；
  - 若当前没有任何插件分组，则展示空态提示，并引导用户点击右上角“添加分组”。

### 5.2 分组管理弹框

弹框展示内容：

- 作者基础信息：头像、昵称、UID（如已有上下文可直接复用）。
- 插件当前全部分组列表（默认直接展示）。
- 右上角“添加分组”按钮。
- 点击“添加分组”后展开创建区：
  - 当前用户“我创建的收藏夹”缓存快照中，尚未绑定为插件分组的候选收藏夹下拉选择；
  - “刷新收藏夹列表”按钮，用于手动刷新该快照；
  - “新建收藏夹并创建分组”入口与表单。
- 每个分组的选中状态：
  - 已选中：使用对勾图标表示当前作者属于该分组；
  - 未选中：使用空心圆图标表示当前作者不属于该分组。

弹框内不展示以下能力：

- 分组别名编辑；
- 分组启用/停用；
- 分组删除；
- 收藏夹隐私切换（新建时固定为私密）。

弹框交互规则：

- 点击未选中的分组：执行“加入分组”。
- 点击已选中的分组：执行“移除分组”。
- 默认不直接展示全部可用收藏夹，避免主视图被“创建入口”挤占。
- 点击右上角“添加分组”按钮后，展开创建区。
- 在创建区内通过“收藏夹下拉选择 + 创建按钮”创建插件分组。
- 弹框打开时不得为了读取收藏夹候选项而自动请求 `nav` / `list-all`；若当前没有收藏夹快照，则展示“尚未加载收藏夹列表”的提示，并提供手动刷新按钮。
- 在创建区内点击“新建收藏夹”后，再展开“新建收藏夹并创建分组”表单。
- 单次点击只影响一个分组，不提供批量提交模式。
- 请求进行中时，对应分组项进入 pending 状态，避免重复提交。
- 创建区的创建按钮与提交按钮都要进入 pending 状态，避免重复提交。
- 操作结果统一通过 toast 反馈，不用整块错误态替换弹框内容。
- 通过弹框新建的插件分组，只创建分组本身，不自动把当前作者加入该新分组。

### 5.3 弹框内创建插件分组

#### 5.3.1 从已有收藏夹创建插件分组

- 数据来源：当前用户“我创建的收藏夹”缓存快照中，排除已被任一插件分组绑定的收藏夹。
- 交互形式：
  - 默认收起；
  - 用户点击右上角“添加分组”后，展示与设置页一致风格的收藏夹下拉框；
  - 同区域提供“刷新收藏夹列表”按钮，点击后显式请求远端并更新快照；
  - 用户从下拉框中选择一个收藏夹，再点击“创建分组”。
- 点击后执行：
  1. 以该收藏夹 `mediaId` 与标题创建新的 `GroupConfig`；
  2. 默认 `enabled = true`；
  3. 默认 `excludeFromUnreadCount = false`；
  4. 不设置别名。
- 创建成功后：
  - 该条目应从“可创建分组的收藏夹列表”中移除；
  - 它应立即出现在“插件分组列表”中；
  - 不自动把当前作者加入该分组；
  - 仍沿用现有“新增分组后立即补一次收藏夹刷新”的后台规则。

#### 5.3.2 新建收藏夹并创建插件分组

- 表单字段：
  - 收藏夹标题（必填）；
  - 不展示别名字段；
  - 不展示公开/私密切换，固定按“私密收藏夹”创建。
- 提交链路：
  1. 调用 Bilibili 新建收藏夹接口，`privacy=1`；
  2. 以新返回的收藏夹 `mediaId` 与标题创建插件分组；
  3. 新分组默认 `enabled = true`、`excludeFromUnreadCount = false`，且不设置别名。
- 成功后：
  - 新分组立即出现在“插件分组列表”中；
  - 新收藏夹应直接回写到本地收藏夹快照，并且不再出现在“可创建分组的收藏夹列表”中；
  - 表单重置；
  - 不自动把当前作者加入该分组；
  - 仍沿用现有“新增分组后立即补一次收藏夹刷新”的后台规则。
- 失败处理：
  - 收藏夹创建失败时，不得继续创建插件分组；
  - 插件分组创建失败时，应保留收藏夹已创建成功这一事实，并明确 toast 提示用户后续可去设置页继续绑定。

## 6. 业务规则

### 6.1 加入分组

#### 6.1.1 视频投稿页

- 若操作入口来自视频投稿页，则直接把当前视频加入目标分组绑定的收藏夹。
- 加入成功后：
  - 立即把该作者写入对应分组缓存的 `authorMids`；
  - 为该分组补一个“收藏夹刷新”请求，让后台后续校正收藏夹标题、作者列表与远端真相。

#### 6.1.2 用户空间 / 头像卡片

- 若操作入口来自用户空间或头像 hover 卡片，则先获取该作者最新公开视频。
- 取值口径：按作者公开视频列表发布时间倒序的第 1 条视频。
- 若存在最新视频，则把该视频加入目标分组绑定的收藏夹。
- 若作者没有任何公开视频，toast 提示“该作者暂无可加入收藏夹的视频”。
- 加入成功后：
  - 立即把该作者写入对应分组缓存的 `authorMids`；
  - 为该分组补一个“收藏夹刷新”请求。

### 6.2 移除分组

- 点击已选中的分组时，把该作者在目标收藏夹中的所有投稿全部移除。
- 这里的“所有投稿”以当前目标收藏夹内命中的该作者视频集合为准，允许一次移除多条。
- 移除成功后：
  - 立即把该作者从对应分组缓存的 `authorMids` 中移除；
  - 为该分组补一个“收藏夹刷新”请求，用于与远端结果重新对齐。

## 7. 缓存与调度器协作

### 7.1 本地缓存即时改写

- 本功能对分组归属的成功操作，允许直接修改本地 `GroupFeedCache.authorMids`。
- 直接改写的目的：
  - 让按钮状态与弹框选中状态立即更新；
  - 避免用户完成操作后仍短时间看到旧状态。

### 7.2 后台校正

- 每次成功的加入/移除操作后，都必须为对应分组追加一次“收藏夹刷新”请求。
- 该刷新请求不要求阻塞当前弹框关闭或 UI 状态更新。
- 调度器负责在后续刷新中重新拉取收藏夹内容并校正作者列表真相。
- 当“加入分组”成功后，若该作者当前还没有完成首轮投稿首页缓存初始化，则必须立即补一个作者投稿初始化请求，行为应与“新增分组后补作者首页缓存”的现有规则保持一致：
  - 无作者缓存：进入 Burst 队列；
  - 有作者缓存但尚无首页抓取时间：进入常规优先队列。

### 7.3 创建分组时的缓存与列表同步

- 通过弹框创建插件分组后，前台应立即更新当前作者弹框内的两类列表：
  - 插件分组列表；
  - 尚未绑定的收藏夹下拉候选项（基于本地快照即时回写，不等待重新请求远端）。
- 该同步以本地成功结果为准，不等待后续刷新任务完成。
- 因为“仅创建分组，不自动加入当前作者”，所以创建成功后：
  - 当前作者的 `grouped` 状态不因此变化；
  - 作者入口按钮文案保持原状，除非该作者原本就属于其他分组。

## 8. API 与消息扩展

本能力需要新增以下链路：

- 将单个视频加入指定收藏夹；
- 将收藏夹中指定作者的全部视频批量移除；
- 读取当前作者在各分组中的归属状态；
- 读取当前用户“尚未绑定为插件分组的收藏夹列表”；
- 新建私密收藏夹；
- 用某个收藏夹直接创建插件分组；
- 从内容脚本向 background 发起“作者分组管理”消息；
- background 成功后回写缓存并提交分组收藏夹刷新任务。

POST 写请求继续遵循项目既有约束：

- 默认由 background 发起；
- 发起前通过 DNR session rule 改写 `Origin`、`Referer`、`Sec-Fetch-Site`；
- 请求完成后立即清理规则；
- 与现有关注/点赞写请求共用同一类来源头改写原则。

新增消息建议：

- `GET_AUTHOR_GROUP_DIALOG_DATA`
  - 返回：
    - 当前作者在已有插件分组中的归属状态；
    - 尚未绑定为插件分组的收藏夹列表（供右上角创建区的下拉框使用）。
- `CREATE_AUTHOR_GROUP_FROM_FOLDER`
  - 入参：`mediaId`、`mediaTitle`
  - 行为：直接创建插件分组，不自动加入作者。
- `CREATE_FOLDER_AND_AUTHOR_GROUP`
  - 入参：`title`、`csrf`、`pageOrigin`、`pageReferer`
  - 行为：先创建私密收藏夹，再创建插件分组，不自动加入作者。

消息返回约束：

- 创建类消息成功后，需要返回：
  - 最新的插件分组列表；
  - 最新的“可创建分组收藏夹下拉候选项”；
  - 当前作者的分组归属状态；
  - 用户可直接展示的 toast 文案。

## 9. 验收标准

1. 在视频投稿页、用户空间、头像 hover 卡片中，都能看到插件注入的作者分组管理按钮。
2. 按钮能正确区分“添加到分组”与“已分组”两种状态。
3. 点击按钮后能打开弹框，并看到全部插件分组与当前作者归属。
4. 在视频投稿页点击某分组加入时，会把当前视频加入对应收藏夹。
5. 在用户空间或头像卡片点击某分组加入时，会把作者最新公开视频加入对应收藏夹；若无公开视频则 toast 提示失败原因。
6. 点击已选中分组移除时，会把该作者在对应收藏夹中的所有投稿移除。
7. 成功的加入/移除操作会立即更新本地缓存，使按钮状态与弹框状态即时变化。
8. 成功的加入/移除操作都会追加一次该分组的收藏夹刷新请求。
9. 弹框默认只展示“已有插件分组”；创建入口位于右上角“添加分组”按钮。
10. 点击“添加分组”后，可通过下拉框选择尚未绑定的收藏夹并创建插件分组。
11. 弹框中能够直接新建私密收藏夹，并同步创建插件分组。
12. 现有分组条目使用对勾/空心圆等图形标记当前归属，而不是使用状态文本。
13. 通过弹框创建插件分组后，不自动把当前作者加入新分组。
14. 弹框内不提供分组别名编辑；别名修改仍需进入设置页。
15. 当用户尚未配置任何分组时，弹框仍可通过右上角“添加分组”入口使用“从已有收藏夹创建分组”与“新建收藏夹并创建分组”能力。
