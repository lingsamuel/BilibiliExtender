# 已阅标记与观看状态 Spec

## 1. 概述

本 Spec 定义“已阅/未读追踪、概览过滤、VideoCard 已阅交互、作者级跨组已阅”的完整行为，目标是：

- 让“红点追踪”与“快速浏览”语义分离，避免临时筛选污染未读基线。
- 保留“已看完”能力，同时提供统一的“已阅”手动标记入口。
- 支持作者级已阅基线跨组共享，减少重复清理未读操作。

## 2. 术语

- **分组已阅时间点（GroupReadMark）**：按分组存储的已阅时间戳。
- **作者级已阅时间点（AuthorReadMark）**：按作者 `mid` 存储、跨分组共享的已阅时间戳。
- **追踪基线（Tracking Baseline）**：用于 unread 计数的真实基线，仅来源于分组/作者已阅，不受概览过滤影响。
- **概览过滤（Overview Filter）**：只影响当前列表展示，不参与 unread 计算。
- **已阅（Reviewed）**：VideoCard 底部作者信息行右侧对勾状态；已阅视频不计入 unread。
- **已看完（Finished）**：播放进度 `playback_position >= 90`，保持原有展示。

## 3. 功能需求

### 3.1 分组已阅时间点

#### 3.1.1 数据模型

```ts
interface GroupReadMark {
  groupId: string;
  // 最多 10 条，按时间倒序（最新在前）
  timestamps: number[];
}
```

- 存储键：`bbe:group-read-marks`。

#### 3.1.2 标记按钮

- 时间流模式：
  - 普通分组：按钮文案为“标记已阅”，写入当前时间戳到当前分组。
  - “全部”分组（虚拟分组）：按钮文案为“标记全部分组为已阅”，对所有启用中的真实分组分别写入当前时间戳。
- 按作者模式：
  - 普通分组：按钮文案为“全部标记已阅”，语义仍为“将当前分组追踪基线推进到当前时间戳”。
  - “全部”分组：按钮文案为“标记全部分组为已阅”，行为同上（批量写入真实分组）。

#### 3.1.3 已阅下拉菜单

下拉菜单在“时间流 / 按作者”两种模式都显示，固定包含两类选项：

1. 追踪选项（参与 unread 基线）
- `全部`
- `N天内`（由 `defaultReadMarkDays` 推导）
- 当前分组历史已阅时间点（倒序）

2. 概览选项（不参与 unread 基线）
- `14天内`
- `30天内`
- `10条`
- `30条`

默认值规则：
- 有历史已阅记录时：默认选中最新一条历史记录。
- 无历史时：默认选中 `N天内`。

#### 3.1.4 过滤语义

当选中追踪选项时：
- `全部`：不过滤。
- `N天内`：使用 grace 时间作为边界。
- 历史时间点：使用该时间点作为边界。

当选中概览选项时：
- `14天内/30天内`：按“当前时间向前 N 天”过滤可见视频。
- `10条/30条`：
  - 按作者模式：每位作者仅保留最新 N 条。
  - 时间流模式：先对每位作者截取最新 N 条，再聚合混排去重。

约束：
- 概览选项只影响展示，不写入追踪基线，不改变分组真实 unread。

### 3.2 作者级跨组已阅时间

#### 3.2.1 数据模型

```ts
interface AuthorReadMark {
  mid: number;
  // 秒级时间戳；不存在表示“默认（未单独设置）”
  readMarkTs?: number;
  updatedAt?: number;
}
```

- 存储键：`bbe:author-read-marks`。
- 作者级配置按 `mid` 全局共享，跨分组生效。

#### 3.2.2 优先级

- 作者级已阅时间点优先于分组已阅时间点。
- 优先级为“绝对覆盖”，与时间大小无关。

#### 3.2.3 交互

- 基于“按作者模式边界竖线”扩展交互：
  - 左键点击竖线（或其可点击热区）：将当前边界写入该作者的作者级已阅时间点。
  - 鼠标悬停时显示可交互反馈（光标/高亮）。
  - 右键点击竖线：清除该作者的作者级已阅时间点，恢复默认（仅使用分组基线）。
- 按作者标题右侧新增“标记已阅”按钮：
  - 点击后将该作者 `readMarkTs` 直接写为当前时间戳（`now`）。
  - 该按钮与竖线交互共存，均写入同一作者级已阅字段。

#### 3.2.4 特殊样式

- 当作者存在“非默认值（已显式设置）”作者级已阅时间点时，竖线必须使用特殊样式（区别于普通分组边界）。
- 特殊样式要求：
  - 在常规粉色主题上增加可辨识标记（如双线、徽点、虚线描边等）。
  - 不改变网格排版与卡片尺寸。
  - 保持与主题变量 `--bbe-theme-color` 一致。

### 3.3 VideoCard 的“已阅”与“已看完”

#### 3.3.1 展示

- 保留原有“已看完”展示逻辑（`playback_position >= 90`）。
- 在 VideoCard 底部作者信息行右侧新增“已阅对勾”按钮：
  - 已阅：绿色勾。
  - 未阅：灰色勾。
  - hover：颜色加深。
- 点击对勾只切换已阅状态，不打开新页面。

#### 3.3.2 状态来源与优先级

“已阅对勾”的最终状态由以下来源合成：

1. 手动覆盖（最高优先级）
- 用户点击对勾后写入显式状态（true/false）。

2. 自动判定（无手动覆盖时生效）
- 点击记录命中（`RECORD_VIDEO_CLICK`）。
- 或播放进度 `playback_position >= 10`。

3. 默认
- 未命中上述条件时为未阅。

补充：
- “已阅”与“已看完”不互斥，可同时存在。
- 已阅视频不计入 unread 统计。

### 3.4 红点与 unread 规则

#### 3.4.1 分组 unread

分组未读数 = 该分组内去重视频中同时满足：
- `pubdate > trackingBaselineTs`
- 且未命中最终“已阅”状态

其中 `trackingBaselineTs` 取值规则：
- 若作者存在作者级已阅时间点：使用作者级时间点（绝对覆盖）。
- 否则使用分组追踪选项对应基线（`全部`/`N天内`/历史时间点）。

#### 3.4.2 概览过滤不参与计数

以下选项不参与 unread 计算：
- `14天内`
- `30天内`
- `10条`
- `30条`

即使用户当前选中了上述任一选项，红点与 Header unread 仍基于“追踪基线”计算。

#### 3.4.3 “全部”分组

- “全部”分组只做聚合展示，不保存独立追踪基线。
- “全部”分组中的“标记全部分组为已阅”会批量写入真实分组，不直接写入虚拟分组自身。

## 4. 存储

### 4.1 存储键

| 键 | 内容 | 说明 |
|---|---|---|
| `bbe:group-read-marks` | `Record<string, GroupReadMark>` | 分组已阅时间点 |
| `bbe:author-read-marks` | `Record<number, AuthorReadMark>` | 作者级跨组已阅时间点 |
| `bbe:clicked-videos` | `Record<string, number>` | 视频点击记录（自动已阅信号） |
| `bbe:video-reviewed-overrides` | `Record<string, boolean>` | 手动对勾覆盖状态 |

### 4.2 清理策略

- `bbe:clicked-videos` 与作者视频缓存联动：视频不再存在于任何作者缓存时清理。
- `bbe:video-reviewed-overrides` 采用同样的“孤儿清理”策略，避免长期累积无效键。

## 5. 消息协议扩展

```ts
| { type: 'MARK_ALL_GROUPS_READ' }
| { type: 'SET_VIDEO_REVIEWED'; payload: { bvid: string; reviewed: boolean } }
| { type: 'GET_VIDEO_REVIEWED_OVERRIDES'; payload: { bvids: string[] } }
| { type: 'SET_AUTHOR_READ_MARK'; payload: { mid: number; readMarkTs: number } }
| { type: 'CLEAR_AUTHOR_READ_MARK'; payload: { mid: number } }
| { type: 'GET_AUTHOR_READ_MARKS'; payload: { mids: number[] } }
```

说明：
- `MARK_GROUP_READ_MARK` 保留用于单分组写入。
- `MARK_ALL_GROUPS_READ` 仅用于“全部”分组工具栏按钮。

## 6. UI 变更

### 6.1 工具栏

```text
时间流-普通分组: [已阅时间点 ▼] [标记已阅]
按作者-普通分组: [已阅时间点 ▼] [全部标记已阅]
时间流/按作者-全部分组: [已阅时间点 ▼] [标记全部分组为已阅]
```

### 6.2 VideoCard

```text
右上角: 已看完标签（保留）
底部作者信息行: [头像] [作者名(链接)+时间] [已阅对勾(可点击)]
```

### 6.3 作者边界竖线

- 竖线支持 hover 反馈、左键设置、右键清除。
- 作者级已阅时间点为非默认值时，竖线使用“特殊样式”。

## 7. 实现要点

- 概览过滤与追踪基线必须分离，避免错误影响红点与 Header unread。
- 作者级已阅优先级为绝对覆盖，不做“取大/取小”比较。
- “已阅对勾”点击事件必须阻断链接跳转，保证“就地切换状态”。
- 所有粉色相关 UI（分隔符、边界线、特殊态）统一复用主题变量，避免硬编码散落。
