# 已阅标记与近期追踪 Spec

## 1. 概述

本 Spec 定义“近期追踪、作者级已阅、VideoCard 已阅交互、未读计数”的完整行为，目标是：

- 让产品心智稳定围绕“查看作者们的近期投稿”展开。
- 让“时间流”和“近期投稿”共享同一套近期基线，避免模式切换后语义漂移。
- 让“全部投稿”成为纯展示模式，不再污染分组红点与 Header unread。
- 保留作者级跨组已阅能力，并补齐明确的撤销路径。

## 2. 术语

- **分组已阅时间点（GroupReadMark）**：按分组存储的自定义“上次看到”时间戳。
- **作者级已阅时间点（AuthorReadMark）**：按作者 `mid` 存储、跨分组共享的已阅时间戳。
- **近期范围设置（Recent Days Setting）**：用户在设置页配置的近期天数，允许任意 `1-30` 整数，默认 `7`。
- **分组近期基线（Group Tracking Baseline）**：当前分组对“时间流 / 近期投稿 / unread”生效的持久化分组级基线，来源可以是 `N天前`，也可以是某个具体时间点。
- **分组“上次看到”时间点（Last Seen Mark）**：用户通过时间流内“上次看到这里”设置出的最新自定义时间点；它和 `N天前` 一样，都属于分组近期基线的一种取值。
- **有效已阅基线（Effective Baseline）**：某作者在某分组下用于“近期投稿”和 unread 计算的最终基线。优先取作者级已阅时间点，否则取分组近期基线。
- **全部投稿筛选（All Posts Filter）**：只影响“全部投稿”当前展示的筛选，不参与 unread 计算。
- **已阅（Reviewed）**：VideoCard 底部作者信息行右侧对勾状态；已阅视频不计入 unread。
- **已看完（Finished）**：播放进度 `playback_position >= 90`，保持原有展示。

## 3. 功能需求

### 3.1 分组级近期追踪

#### 3.1.1 数据模型

```ts
interface GroupReadMark {
  groupId: string;
  // 最多 10 条，按时间倒序（最新在前）
  // 精度统一到分钟（秒归零）
  timestamps: number[];
}
```

- 存储键：`bbe:group-read-marks`。
- UI 只展示最新一个分组“上次看到”时间点，不展示完整历史列表。
- 历史时间点仍可保留在存储中，用于“撤销上次看到”时回退到上一个自定义时间点。

#### 3.1.2 模式与基线关系

- `时间流` 与 `近期投稿` 共享同一套分组级近期设置：
  - 一个持久化的分组近期基线；
  - 该基线可取 `N天前`，也可取某个具体时间点。
- `全部投稿` 独立于近期追踪：
  - 其筛选只影响展示；
  - 不写入分组近期基线；
  - 不参与分组 unread 与 Header unread 计算。
- `时间流` 中的 `全部` 只是临时查看状态：
  - 不写入分组持久化状态；
  - 不改写“近期投稿”的结果；
  - 不改变红点与 Header unread。

#### 3.1.3 工具栏与按钮

- 时间流：
  - 普通分组：`[近期范围 ▼] [撤销上次看到] [标记已阅]`
  - 当不存在分组“上次看到”时间点时，不显示 `撤销上次看到`
  - “全部”分组：`[近期范围 ▼] [标记全部分组为已阅]`
- 近期投稿：
  - 普通分组：`[近期范围 ▼] [全部标记已阅]`
  - “全部”分组：`[近期范围 ▼] [标记全部分组为已阅]`
- 全部投稿：
  - 任意分组：`[按更新时间倒序] [全部投稿筛选 ▼]`
  - 不显示分组级“标记已阅”按钮

补充：
- 时间流 / 近期投稿的“近期范围 ▼”包含：
  - 当前设置页定义的 `N天前`
  - 最新的分组“上次看到”时间点（若存在）
- 只有时间流额外支持临时 `全部` 选项。
- 即使当前分组已存在具体时间点，`N天前` 也必须继续显示，允许用户在“具体时间点”与“按 N 天追踪”之间显式切换。

#### 3.1.4 默认值与持久化

- 时间流 / 近期投稿：
  - 默认使用设置页中的 `N天前`（默认 `7天前`）；
  - 用户显式切换到 `N天前` 后，按分组持久化，并作为有效基线参与 unread；
  - 用户通过“上次看到这里”写入具体时间点后，该时间点也按分组持久化，并作为有效基线参与 unread；
  - 时间流中临时切换到 `全部` 不持久化。
- 全部投稿：
  - 默认选中 `全部`；
  - 筛选状态按分组持久化；
  - “按更新时间倒序”勾选状态按分组持久化；
  - 不影响近期追踪。

#### 3.1.5 时间流“上次看到这里”与撤销

- 时间流中的“上次看到这里”分割线支持任意已阅时间点，不再仅限“日边界（00:00）”。
- 命中任意分组“上次看到”时间点时，分割线插入规则：
  - 在混排结果中定位第一条“严格早于已阅时间点”的视频；
  - 在该视频之前显示“上次看到这里”横向分割线；
  - 若第一条视频本身已早于已阅时间点，则分割线显示在列表顶部。
- 即使时间流当前临时切换到 `全部`，该分割线仍然显示，用于提示最近一次看到的位置。
- 时间流视图保持“防突变”展示策略：
  - 不使用“严格按已阅时间硬截断”的展示方式；
  - 展示下界至少覆盖当前设置页定义的近期窗口（默认 `7天内`）；
  - 若已阅时间点早于该下界，则展示范围继续向更早延展。
- `撤销上次看到` 规则：
  - 仅撤销最近一次由“上次看到这里”写入的分组级自定义时间点；
  - 若撤销后仍有更早的自定义时间点，则继续使用最新一条作为有效基线；
  - 若撤销后已无自定义时间点，则恢复到当前持久化的 `N天前` 基线；
  - 撤销结果会同步影响时间流与近期投稿。

#### 3.1.6 时间流“卡片间设置已阅”交互

- 在时间流视图中，鼠标 hover 到相邻 VideoCard 之间时，显示半透明粉色竖线虚影（仅作为交互提示）。
- 点击该竖线虚影后，将当前分组“上次看到”时间点设置到“这两条视频之间”的时间点：
  - 常规：`olderTs + 1`
  - 同秒等不可切分场景：退化为 `newerTs`
- 设置成功后，分割展示使用“上次看到这里”横向样式，不保留竖线作为持久分割元素。

#### 3.1.7 分钟精度与去重

- 分组“上次看到”时间点写入时统一归一化到分钟精度（秒归零）。
- 若同一分组在同一分钟内重复写入，不新增时间点（按分钟去重）。

### 3.2 作者级跨组已阅时间

#### 3.2.1 数据模型

```ts
interface AuthorReadMark {
  mid: number;
  // 当前最新作者级已阅时间点；为空表示“默认（未单独设置）”
  readMarkTs?: number;
  // 最多保留 10 条，按时间倒序（最新在前）
  readMarkTimestamps?: number[];
  updatedAt?: number;
}
```

- 存储键：`bbe:author-read-marks`。
- 作者级配置按 `mid` 全局共享，跨分组生效。
- 历史栈最多保留 10 条，按时间倒序排列。

#### 3.2.2 优先级

- 作者级已阅时间点优先于分组级有效基线。
- 优先级为“绝对覆盖”，与时间大小无关。

#### 3.2.3 交互

- 基于按作者模式中的边界竖线扩展交互：
  - 左键点击竖线（或其可点击热区）：将当前边界写入该作者的作者级已阅时间点；
  - 鼠标悬停时显示可交互反馈（光标/高亮）；
  - 右键点击竖线：清空该作者整条作者级已阅历史栈，恢复默认（仅使用分组有效基线）。
- 按作者标题右侧新增作者级操作按钮：
  - `标记已阅` 按钮始终显示，点击后将当前时间戳（`now`）压入该作者的作者级已阅历史栈；
  - 当作者存在显式作者级已阅时间点时，额外显示 `撤销已阅` 按钮；
  - 点击 `撤销已阅` 后只回退最近一次作者级已阅写入；
  - 若撤销后栈内仍有更早时间点，则继续使用最新一条；
  - 若撤销后栈空，则恢复为使用分组有效基线。
- 以上交互在“近期投稿”和“全部投稿”中保持一致，且都写入同一作者级字段。

#### 3.2.4 特殊样式

- 当作者存在“非默认值（已显式设置）”作者级已阅时间点时，竖线必须使用特殊样式（区别于普通分组边界）。
- 竖线位置必须位于“前后两个视频卡片之间”的几何中线，不允许贴靠任一侧卡片。
- 特殊样式要求：
  - 在常规粉色主题上增加可辨识标记（如双线、徽点、虚线描边等）；
  - 不改变网格排版与卡片尺寸；
  - 保持与主题变量 `--bbe-theme-color` 一致。

### 3.3 VideoCard 的“已阅”与“已看完”

#### 3.3.1 展示

- 保留原有“已看完”展示逻辑（`playback_position >= 90`）。
- 在 VideoCard 底部作者信息行右侧新增“已阅对勾”按钮：
  - 已阅：绿色勾；
  - 未阅：灰色勾；
  - hover：颜色加深。
- 点击对勾只切换已阅状态，不打开新页面。

#### 3.3.2 状态来源与优先级

“已阅对勾”的最终状态由以下来源合成：

1. 手动覆盖（最高优先级）
- 用户点击对勾后写入显式状态（true/false）。

2. 自动判定（无手动覆盖时生效）
- 点击记录命中（`RECORD_VIDEO_CLICK`）。
- 或播放进度 `playback_position >= 10`。

3. 默认
- 未命中上述条件时为未阅。

补充：
- “已阅”与“已看完”不互斥，可同时存在。
- 已阅视频不计入 unread 统计。

### 3.4 红点与 unread 规则

#### 3.4.1 分组 unread

分组未读数 = 该分组内去重视频中同时满足：
- `pubdate > trackingBaselineTs`
- 且未命中最终“已阅”状态

其中 `trackingBaselineTs` 取值规则：
- 若作者存在作者级已阅时间点：使用作者级时间点（绝对覆盖）。
- 否则使用当前分组持久化的分组近期基线：
  - 若当前基线为某个具体时间点，则使用该时间点；
  - 若当前基线为 `N天前`，则使用按当前设置页天数计算出的时间边界。

#### 3.4.2 不参与计数的展示筛选

以下展示状态不参与 unread 计算：
- 时间流中的临时 `全部`
- 全部投稿中的 `全部`
- 全部投稿中的 `7天内`
- 全部投稿中的 `14天内`
- 全部投稿中的 `30天内`
- 全部投稿中的 `10条`
- 全部投稿中的 `30条`

即使用户当前选中了上述任一选项，红点与 Header unread 仍基于“有效已阅基线”计算。

#### 3.4.3 “全部”分组

- “全部”分组只做聚合展示，不保存独立追踪基线。
- “全部”分组中的“标记全部分组为已阅”会批量写入真实分组，不直接写入虚拟分组自身。

## 4. 存储

### 4.1 存储键

| 键 | 内容 | 说明 |
|---|---|---|
| `bbe:group-read-marks` | `Record<string, GroupReadMark>` | 分组“上次看到”时间点历史 |
| `bbe:author-read-marks` | `Record<number, AuthorReadMark>` | 作者级跨组已阅时间点 |
| `bbe:clicked-videos` | `Record<string, number>` | 视频点击记录（自动已阅信号） |
| `bbe:video-reviewed-overrides` | `Record<string, boolean>` | 手动对勾覆盖状态 |

### 4.2 清理策略

- `bbe:clicked-videos` 与作者视频缓存联动：视频不再存在于任何作者缓存时清理。
- `bbe:video-reviewed-overrides` 采用同样的“孤儿清理”策略，避免长期累积无效键。

## 5. 消息协议扩展

```ts
| { type: 'MARK_ALL_GROUPS_READ' }
| { type: 'UNDO_GROUP_READ_MARK'; payload: { groupId: string } }
| { type: 'SET_VIDEO_REVIEWED'; payload: { bvid: string; reviewed: boolean } }
| { type: 'GET_VIDEO_REVIEWED_OVERRIDES'; payload: { bvids: string[] } }
| { type: 'SET_AUTHOR_READ_MARK'; payload: { mid: number; readMarkTs: number } }
| { type: 'CLEAR_AUTHOR_READ_MARK'; payload: { mid: number } }
| { type: 'GET_AUTHOR_READ_MARKS'; payload: { mids: number[] } }
```

说明：
- `MARK_GROUP_READ_MARK` 保留用于单分组写入。
- `MARK_ALL_GROUPS_READ` 仅用于“全部”分组工具栏按钮。
- `UNDO_GROUP_READ_MARK` 用于时间流中的 `撤销上次看到`。

## 6. UI 变更

### 6.1 工具栏

```text
模式按钮: [时间流] [近期投稿] [全部投稿]
时间流-普通分组: [近期范围 ▼] [撤销上次看到] [标记已阅]
近期投稿-普通分组: [近期范围 ▼] [全部标记已阅]
时间流/近期投稿-全部分组: [近期范围 ▼] [标记全部分组为已阅]
全部投稿（任意分组）: [全部投稿筛选 ▼]
```

### 6.2 VideoCard

```text
右上角: 已看完标签（保留）
底部作者信息行: [头像] [作者名(链接)+时间] [已阅对勾(可点击)]
```

### 6.3 作者边界竖线

- 竖线支持 hover 反馈、左键设置、右键清除，并保持在两张卡片中间。
- 作者级已阅时间点为非默认值时，竖线使用“特殊样式”。

## 7. 实现要点

- “时间流”和“近期投稿”必须共享同一套分组级近期设置。
- 时间流中的临时 `全部` 不得写回持久化状态，也不得影响 unread。
- 作者级已阅优先级为绝对覆盖，不做“取大/取小”比较。
- “撤销上次看到”只回退最近一次分组级自定义时间点，不影响作者级已阅。
- 作者标题上的 `撤销已阅` 只回退最近一次作者级已阅写入，不清空整条作者历史；清空仍由边界线右键负责。
- “已阅对勾”点击事件必须阻断链接跳转，保证“就地切换状态”。
- 分组“上次看到”时间点必须采用“分钟精度 + 同分钟去重”规则。
- 所有粉色相关 UI（分隔符、边界线、特殊态）统一复用主题变量，避免硬编码散落。
